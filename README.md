# Hearst-Fujingaho-JP 運用ガイド
本ガイドは2025/12現在まだドラフトです

このドキュメントは、Hearst Fujingahoおよび、Hearst Digital Japan（以下 HDJ）が運用する GitHub Organization に参加する
すべてのメンバー（社員・業務委託・ベンダー）向けの共通ルールを定めたものです。

---

## 1. この Organization の目的

* HDJ が開発・運用するシステムおよび関連コードを管理する
* 社内外メンバーが安全かつ効率的に協業できる環境を提供する
* セキュリティ・監査・属人化防止を重視した運用を行う

---

## 2. 対象者

* HDJ 社員
* HDJ と契約関係にある業務委託・外部ベンダー
* 一時的に参加するプロジェクトメンバー

※ 本 Organization への参加は、管理者の承認が必要です。

---

## 3. GitHubアカウントに関するルール

### 3.1 アカウント種別

* GitHubアカウントは **原則として個人アカウントを利用**してください
* 会社専用・共有アカウントの作成および利用は非推奨です
* Bot / CI / 自動化用途は例外として許可します

### 3.2 表示名（Display name）

以下の形式を推奨します：

* 社員は下記としてください

  ```
  First Name Last Name(HDJ)
  ```

  例：Taku Matsumoto(HDJ)

* 社員以外は下記を推奨します

  ```
  First Name Last Name(会社名)
  ```

  例：Tar Yamada(ABC Corp)

※ Display name は GitHub 全体で共通のため、Organization ごとに変更はできません。

### 3.3 メールアドレス

* 社員： hearst.co.jp ドメインのメールアドレスを登録してください
* ベンダー： 業務用メールアドレスを登録してください
* フリーメールアドレスのみの利用は不可とします

### 3.4 二要素認証（2FA）

* **2FA は必須**です
* 有効化されていない場合、Organization から削除されることがあります

---

## 4. 権限管理の方針

* 個人に直接 Repository 権限を付与することは避けます
* 権限は **Team 単位**で付与します
* Role（Read / Write / Maintain / Admin）は最小権限を原則とします

---

## 5. 開発・レビューの基本ルール

* 直接 main / master ブランチへ push しないでください
* Pull Request を作成し、レビューを経てマージします
* ~~CODEOWNERS に指定されたレビューを必須とします~~

---

## 6. Commit / Pull Request の注意点

* Commit author は個人が特定できる名前を使用してください
* 業務コードは HDJ Organization の Repository にのみ commit してください
* 機密情報（トークン・パスワード等）を commit しないでください

---
